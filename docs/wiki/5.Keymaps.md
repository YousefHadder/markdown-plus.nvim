# Keymaps Reference

Default mappings use the `<localleader>` namespace in v2.0.

## List Management

### Insert mode

| Keymap | Description |
|---|---|
| `<CR>` | Auto-continue list item / split list content |
| `<A-CR>` | Continue current list item content on next line |
| `<Tab>` | Indent list item |
| `<S-Tab>` | Outdent list item |
| `<BS>` | Smart backspace on list markers |
| `<C-t>` | Toggle checkbox |

| Keymap | Description |

### List type toggle

Toggle the current line or a visual selection between list types. Plain text gains a marker; a line already of the target type is cleared; a line of a different type is converted. Ordered types are renumbered automatically.

| Keymap | List type | Marker |
|---|---|---|
| `<localleader>ltu` | Unordered | `-` |
| `<localleader>ltt` | Task | `- [ ]` |
| `<localleader>ltn` | Numbered | `1.` |
| `<localleader>ltN` | Numbered paren | `1)` |
| `<localleader>ltl` | Lowercase letter | `a.` |
| `<localleader>ltL` | Uppercase letter | `A.` |
| `<localleader>ltp` | Lowercase paren | `a)` |
| `<localleader>ltP` | Uppercase paren | `A)` |
| `<localleader>ltc` | Clear | strip markers (plain text) |

All mappings work in both normal and visual mode. Because these are real buffer-local mappings, `which-key.nvim` shows the menu automatically after `<localleader>lt`. The second key must be pressed within `'timeoutlen'`.

If you prefer a picker that waits indefinitely for the type key, bind the picker `<Plug>` yourself:

```lua
vim.keymap.set({ "n", "x" }, "<localleader>lt",
  "<Plug>(MarkdownPlusToggleListPick)", { buffer = true })
```
