# Usage Examples

The plugin automatically activates when you open a markdown file (`.md` extension). All features work seamlessly with Neovim's built-in functionality.

> [!TIP]
> For v2.0 mapping changes and migration steps, see the [v2.0 Migration Guide](https://github.com/YousefHadder/markdown-plus.nvim/wiki/9.v2.0-Migration-Guide).

## List Management Examples

### Auto-continue Lists

```markdown
- Type your first item and press Enter
- The next item is automatically created ⬅️ (cursor here)
```

### Split List Content

```markdown
- This is some content in| the middle
  Press Enter splits at cursor:
- This is some content in
- the middle ⬅️ (cursor here, new item created with remaining content)
```

### Continue Content Without New Bullet

```markdown
- This is a longer list item that|
  Press Alt+Enter to continue on next line:
- This is a longer list item that
  continues here ⬅️ (same item, no new bullet)
```

### Checkbox Lists

```markdown
- [ ] Press Enter after this unchecked item
- [ ] Next checkbox item is created ⬅️ (cursor here)

1. [x]
2. [ ]
```

### Toggle Checkboxes

```markdown
Position cursor on a list item and press <localleader>mx:

- Regular item       ← Press <localleader>mx
- [ ] Regular item   ← Press <localleader>mx again
- [x] Regular item   ← Press <localleader>mx again (cycles back to unchecked)

Works with all list types:
1. Item              → 1. [ ] Item → 1. [x] Item
a. Item              → a. [ ] Item → a. [x] Item

Visual mode - select multiple lines and press <localleader>mx:
- [ ] Task 1         → - [x] Task 1
- [x] Task 2         → - [ ] Task 2
- Regular            → - [ ] Regular

Insert mode - press Ctrl+T while on a list item:
- [ ] Todo           → - [x] Todo (cursor stays in place)
```

### Checkbox Completion Timestamps

When enabled in configuration, checking a task automatically adds a completion timestamp:

```markdown
Enable with:
  list = { checkbox_completion = { enabled = true } }

Then toggle checkboxes as usual:
- [ ] Buy groceries     → - [x] Buy groceries ✅ 2025-12-28
- [x] Call mom ✅ 2025-12-28  → - [ ] Call mom  (timestamp removed on uncheck)

Works with all list types:
1. [ ] First task       → 1. [x] First task ✅ 2025-12-28
a. [ ] Letter task      → a. [x] Letter task ✅ 2025-12-28

Different formats available (set via format option):
- [x] Task ✅ 2025-12-28                    (emoji - default)
- [x] Task <!-- completed: 2025-12-28 -->  (comment - hidden)
- [x] Task [completion:: 2025-12-28]       (dataview - Obsidian)
- [x] Task (completed: 2025-12-28)         (parenthetical)
```

### Normal Mode List Creation

```markdown
- Position cursor on this list item
- Press 'o' to create next item ⬅️ (new item appears below)
- Press 'O' to create previous item ⬅️ (new item appears above)

1. Works with ordered lists too
2. Press 'o' to create item 3 below ⬅️

```

### Configurable Marker-to-Content Whitespace

By default, one space separates a list marker from its content (`"single"` mode). Set
`list.whitespace = "shiftwidth"` to pad content to a fixed-width column block instead,
keeping continuation lines visually aligned:

```lua
require("markdown-plus").setup({
  list = {
    whitespace = "shiftwidth",  -- "single" (default) | "shiftwidth"
    whitespace_width = 4,       -- integer ≥ 1; block width (default 4)
  },
})
```

```markdown
-- "single" mode (default, standard Markdown):
- Item content
  continuation line

1. Item content
   continuation line

-- "shiftwidth" mode (whitespace_width = 4):
-   Item content      ← content starts at indent 4
    continuation      ← continuation naturally aligns

1.  Item content      ← content starts at indent 4
    continuation      ← continuation naturally aligns
```

> [!NOTE]
> Markdown formatters that enforce single-space list markers (e.g. Prettier) will override
> this setting on format/save.

### Toggle List Type

Press `<localleader>lt` followed by a type key to switch the current line or a visual selection between list
types. Pressing the same key again when the line already matches the target converts it back to plain text.

| Key | Type |
|---|---|
| `u` | Unordered (`-`) |
| `t` | Task (`- [ ]`) |
| `n` | Ordered (`1.`) |
| `N` | Ordered paren (`1)`) |
| `l` | Lowercase letter (`a.`) |
| `L` | Uppercase letter (`A.`) |
| `p` | Lowercase letter paren (`a)`) |
| `P` | Uppercase letter paren (`A)`) |
| `c` | Clear (remove markers) |

```markdown
-- Convert a plain paragraph line to an unordered list:
Some text here       → press <localleader>ltu → - Some text here

-- Toggle an ordered list to a task list (visual selection):
1. First item        → select lines, press <localleader>ltt → - [ ] First item
2. Second item                                               → - [ ] Second item

-- Toggle off: pressing the same key reverts to plain text:
- Some item          → press <localleader>ltu → Some item

-- Ordered types get renumbered automatically:
- Alpha              → press <localleader>ltn → 1. Alpha
- Beta                                        → 2. Beta
- Gamma                                       → 3. Gamma
```

> [!TIP]
> which-key users see the full menu after typing `<localleader>lt`. To use an interactive one-key picker
> instead of nested keymaps, bind `<Plug>(MarkdownPlusToggleListPick)` to `<localleader>lt`:
> ```lua
> vim.keymap.set({"n","x"}, "<localleader>lt", "<Plug>(MarkdownPlusToggleListPick)")
> ```

### Header Navigation

```markdown
Use ]] and [[ to jump between headers quickly:
# Main Title       ← Press ]] to jump here
Content
## Section 1       ← Press ]] to jump here
Content
### Subsection    ← Press ]] to jump here
Content
Press [[ to jump backwards
```

### Promote/Demote Headers

```markdown
Position cursor on any header and adjust its level:
### Subsection    ← Press <localleader>h+ → ## Subsection (promoted)
## Section        ← Press <localleader>h- → ### Section (demoted)
```

### Convert to Header

```markdown
Position cursor on any line:
Regular text      ← Press <localleader>h2 → ## Regular text
Already header    ← Press <localleader>h4 → #### Already header
```

### Generate Table of Contents

```markdown
# My Document

Press <localleader>ht to generate TOC:

<!-- TOC -->

## Table of Contents

- [Section 1](#section-1)
  - [Subsection 1.1](#subsection-1-1)
- [Section 2](#section-2)

<!-- /TOC -->

## Section 1
...
```

**Note:** The TOC is wrapped in HTML comment markers `<!-- TOC -->` and `<!-- /TOC -->`. This prevents duplicate TOCs from being created if you press `<localleader>ht` again. To update an existing TOC, use `<localleader>hu` instead.

### Update TOC

```markdown
After adding/removing/renaming headers:
1. Press <localleader>hu to update the TOC
2. All links are regenerated automatically
3. The content between <!-- TOC --> and <!-- /TOC --> is replaced
```

### Follow TOC Links

```markdown
## Table of Contents

- [Getting Started](#getting-started)  ← Position cursor here
- [API & SDK](#api--sdk)
- [Q&A](#qa)

Press gd to jump directly to that header!

## Getting Started    ← You jump here instantly!
```

### TOC Window (Navigable)

Open an interactive Table of Contents window to browse and navigate your document structure:

```vim
:Toc          " Open TOC in vertical split
:Toch         " Open TOC in horizontal split
:Toct         " Open TOC in new tab
<localleader>hT    " Default keymap to toggle TOC
```

**Features:**
- **Progressive disclosure**: Shows H1-H2 initially, expand on demand
- **Fold/unfold navigation**: `l` to expand, `h` to collapse
- **Color-coded levels**: Each header level has distinct color
- **Visual markers**: `▶` (collapsed), `▼` (expanded)
- **Jump to headers**: Press `<Enter>` on any header
- **Help popup**: Press `?` for keyboard shortcuts

**Example:**
```
Initial View (depth 1-2):
[H1] Main Title
▶ [H2] Getting Started
▶ [H2] Features
▶ [H2] Contributing

After pressing 'l' on "Getting Started":
[H1] Main Title
▼ [H2] Getting Started
    ▶ [H3] Installation
    ▶ [H3] Configuration
▶ [H2] Features
▶ [H2] Contributing
```

**Keymaps (inside TOC):**
- `l` - Expand header to show children
- `h` - Collapse header or jump to parent
- `<Enter>` - Jump to header in source buffer
- `q` - Close TOC window
- `?` - Show help popup

### Smart Paste

Smart paste automatically converts a URL from your clipboard into a markdown link by fetching the page title asynchronously. This feature is opt-in and disabled by default.

**Enable in configuration:**

```lua
require("markdown-plus").setup({
  links = {
    smart_paste = {
      enabled = true,
      timeout = 5,  -- fetch timeout in seconds (1–30)
    },
  },
})
```

**Usage:**

```markdown
1. Copy a URL to your system clipboard (e.g., https://github.com)
2. Place cursor where you want the link inserted
3. Press <localleader>mp

Immediate: [⏳ Loading...](https://github.com) ← placeholder inserted
After fetch: [GitHub](https://github.com)       ← updated with page title
```

When the title fetch fails or times out, you will be prompted to enter a title manually. If you cancel the prompt, the placeholder is replaced with the raw URL.

**Requirements:**

- `curl` must be installed
- Neovim 0.11+ (plugin requirement; includes `vim.system()` async API)
Run `:checkhealth markdown-plus` to verify both are available — it will report a dedicated **Smart Paste** section when the feature is enabled.

### All Callout Types

GitHub renders these with distinct colors and icons:

```markdown
> [!NOTE]
> Useful information that users should know, even when skimming content.

> [!TIP]
> Helpful advice for doing things better or more easily.

> [!IMPORTANT]
```

### Custom Callout Types


### Insert Mode Navigation

```markdown
Navigate table cells in insert mode with Alt+hjkl:

| Name  | Age | City     |
|-------|-----|----------|
| Alice | 25  | New York | ← cursor here
| Bob   | 30  | LA       |

Press <A-l> (Alt+l) to move right:
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 25  | New York |
| Bob   | 30  | LA       | ← cursor moves here
                ^

Press <A-j> (Alt+j) to move down:
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 25  | New York |
| Bob   | 30  | LA       | ← cursor moves down
          ^

Wrapping behavior (circular navigation):
- <A-l> at last column wraps to first column
- <A-h> at first column wraps to last column
- <A-j> at last row wraps to header row
- <A-k> at header row wraps to last data row

Falls back to arrow keys when not in a table.
```

### Cell Operations

```markdown
Toggle cell alignment with <localleader>ta:
| Left | Center | Right |
|------|--------|-------|
| A    | B      | C     |
      ↑ cursor here, press <localleader>ta

→ Cycles through:
| Left | Center | Right |
|:-----|--------|-------| (left-aligned)
|:-----|:------:|-------| (center-aligned)
|:-----|:------:|------:| (right-aligned)
|------|--------|-------| (back to left)

Clear cell content with <localleader>tx:
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 25  | New York |
              ↑ cursor here, press <localleader>tx

→
| Name  | Age | City     |
|-------|-----|----------|
| Alice |     | New York | (content cleared, structure intact)
```

### Multi-line Cell Content

GFM tables forbid real newlines inside cells, but `<br>` is valid HTML and is the canonical way to force a visual line break that renders correctly on GitHub. The plugin provides four explicit commands plus a popup editor.

**Insert a `<br>` at cursor with `<localleader>tb`:**

```markdown
| Name  | Notes                       |
|-------|-----------------------------|
| Alice | builds plugins              |
                       ↑ cursor here, press <localleader>tb

→
| Name  | Notes                       |
|-------|-----------------------------|
| Alice | builds <br>plugins          |
```

Pure text edit — no reformat. Run `<localleader>tf` after if you want re-alignment.

**Wrap a long cell with `<localleader>tw`:**

```markdown
| Description                                 | S |
|---------------------------------------------|---|
| The quick brown fox jumps over the lazy dog | x |

Press <localleader>tw, enter width 20 →

| Description                                       | S |
|---------------------------------------------------|---|
| The quick brown fox<br>jumps over the<br>lazy dog | x |
```

Width is read from `table.max_column_width` if set; otherwise the command prompts. Word boundaries are respected — a single word longer than the cap stays intact on its own line. Inline-code spans (backtick-fenced text) are treated as atomic so wrapping never splits `` `foo bar` `` mid-span. Running the command twice at the same width is a no-op.

**Unwrap a cell with `<localleader>tW`:**

```markdown
| Description                            | S |
|----------------------------------------|---|
| one<br>two<br/>three<br />four         | x |

Press <localleader>tW →

| Description        | S |
|--------------------|---|
| one two three four | x |
```

All three `<br>` variants are recognised (case-insensitive). Idempotent.

**Edit a cell in a popup with `<localleader>te`:**

```markdown
| Description         | S |
|---------------------|---|
| one<br>two<br>three | x |
        ↑ cursor here, press <localleader>te

Floating popup opens with:
  one
  two
  three

Edit freely with real newlines and full Vim editing. Save with <CR>
or :w, cancel with q or <Esc><Esc>. On save the cell becomes:

| Description                      | S |
|----------------------------------|---|
| edited<br>content<br>with breaks | x |
```

A single `u` reverts the popup save. The cursor is restored to the same cell even if widths shift.

**Keep tables visually compact (`width_mode = "segment"`):**

By default the formatter measures the literal cell width including `<br>` tokens, which can make `<br>`-containing tables look stretched in the source buffer. Set `table.width_mode = "segment"` to measure each `<br>`-split segment instead — the rendered GitHub output stays identical but the source stays compact.

**Auto-wrap on every format (`auto_wrap`):**

Set both `table.auto_wrap = true` and `table.max_column_width = N` in `setup()` to have `<localleader>tf` (and every other operation that reformats) auto-inject `<br>` for any cell whose widest segment exceeds the cap. To exempt a specific table, place `<!-- markdown-plus: no-wrap -->` on the line immediately above it.

### Row and Column Movement

```markdown
Move row up with <localleader>tmk:
| Name  | Age |
|-------|-----|
| Alice | 25  |
| Bob   | 30  | ← cursor here
| Carol | 35  |

→
| Name  | Age |
|-------|-----|
| Bob   | 30  | (moved up)
| Alice | 25  |
| Carol | 35  |

Move row down with <localleader>tmj:
| Name  | Age |
|-------|-----|
| Alice | 25  | ← cursor here
| Bob   | 30  |
| Carol | 35  |

→
| Name  | Age |
|-------|-----|
| Bob   | 30  |
| Alice | 25  | (moved down)
| Carol | 35  |

Move column left with <localleader>tmh:
| Name  | Age | City |
|-------|-----|------|
| Alice | 25  | NYC  |
              ↑ cursor here

→
| Age | Name  | City |
|-----|-------|------|
| 25  | Alice | NYC  | (Age column moved left)

Move column right with <localleader>tml:
| Name  | Age | City |
|-------|-----|------|
| Alice | 25  | NYC  |
        ↑ cursor here

→
| Age | Name  | City |
|-----|-------|------|
| 25  | Alice | NYC  | (Name column moved right)
```

### Advanced Operations

```markdown
Transpose table with <localleader>tt:
| Name  | Age | City        |
|-------|-----|-------------|
| Alice | 30  | New York    |
| Bob   | 25  | Los Angeles |
↑ cursor anywhere in table, press <localleader>tt, confirm

→
| Name | Alice    | Bob         |
|------|----------|-------------|
| Age  | 30       | 25          |
| City | New York | Los Angeles |
(Rows become columns, columns become rows)

Sort by column ascending with <localleader>tsa:
| Name  | Age | City        |
|-------|-----|-------------|
| Bob   | 30  | Los Angeles |
| Alice | 25  | New York    |
| Carol | 35  | Chicago     |
        ↑ cursor in Age column, press <localleader>tsa, confirm

→
| Name  | Age | City        |
|-------|-----|-------------|
| Alice | 25  | New York    | (sorted by age ascending)
| Bob   | 30  | Los Angeles |
| Carol | 35  | Chicago     |

Sort by column descending with <localleader>tsd:
| Product | Price |
|---------|-------|
| Apple   | 1.50  |
| Banana  | 0.75  |
| Orange  | 2.00  |
           ↑ cursor in Price column, press <localleader>tsd, confirm

→
| Product | Price |
|---------|-------|
| Orange  | 2.00  | (sorted by price descending)
| Apple   | 1.50  |
| Banana  | 0.75  |

Convert table to CSV with <localleader>tvx:
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 25  | New York |
| Bob   | 30  | LA       |
↑ cursor anywhere in table, press <localleader>tvx

→
Name,Age,City
Alice,25,New York
Bob,30,LA
(Table replaced with CSV format)

Convert CSV to table with <localleader>tvi:
Name,Age,City
Alice,25,New York
Bob,30,LA
↑ cursor on any CSV line, press <localleader>tvi

→
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 25  | New York |
| Bob   | 30  | LA       |
(CSV converted to formatted table)

Handles quoted CSV fields with commas:
"Last, First",Age,"City, State"
"Smith, John",30,"New York, NY"

→
| Last, First | Age | City, State  |
|-------------|-----|---------------|
| Smith, John | 30  | New York, NY |
```

### Edge Cases

```markdown
Tables handle various edge cases:

Empty cells:
| Header 1 | Header 2 |
|----------|----------|
|          | Data     |
| Data     |          |

Special characters:
| Name      | Symbol |
|-----------|--------|
| Greater   | >      |
| Less      | <      |
| Pipe      | \|     |

Unicode:
| English | Japanese | Emoji |
|---------|----------|-------|
| Hello   | こんにちは | 👋    |
| World   | 世界     | 🌍    |

Malformed tables (normalized automatically):
| No closing pipe
| Missing separator
    →
| No closing pipe    |
|--------------------|
| Missing separator  |
```

### Smart Features

```markdown
**Header Protection:**
Cannot delete header row or separator row. Operations protect table integrity.

**Minimum Constraints:**
- Cannot delete the last column
- Cannot delete the only data row
- Maintains at least one header + separator + one data row

**Smart Cursor Positioning:**
After all operations, cursor is automatically positioned in the most
logical cell (usually first cell of new/modified row/column).
```
